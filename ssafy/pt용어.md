devsecops: 개발 운영 보안, 개발 수명 주기의 모든 단계에 보안을 통합하는 접근 방식

end to end: 룰베이스드 방식을 벗어나 모든 과정을 단일 신경망으로 학습하여 빠르게 계산

human in the loop: 인간이 자동화 시스템의 운영 및 의사결정에 적극적으로 참여

sLLM: 대규모 언어 모델을 경량화 하드웨어 요구사항 낮고 비용과 에너지 절감 효과

하이드레이션 에러: 서버 html과 클라이언트 결과가 일치하지 않을 때 발생하는 오류 ssr 방식에서 나옴

제로 트러스트: 아무도 아무것도 신뢰하지 않는다는 원칙, 사용자 장치 애플리케이션의 접근을 지속적으로 인증하고 검증
- 해결방안: SaaS형 클라우드 보안 서비스 SECaaS를 도입 

제로데이: 소프트웨어 제조사가 아직 모르는 보안 취약점을 이용해 이루어지는 공격

RASP: 런타임 시점에 의심스러운 함수 호출이나 파일 접근이 감지되면 즉시 차단하는 기술

뜬금 없어도 기술자의 입장에서 문제를 찾아야 함

트래픽 스파이크: 서버 다운이나 CDN 비용 폭탄을 맞게됨 

엣지 컴퓨팅: 중앙 집중식 클라우드나 데이터 센터에만 의존하지 않고 데이터가 생성되는 지점 근처에서 컴퓨팅 작업을 수행

엣지 서버: 네트워크의 경계에 위치하여 데이터 처리 및 서비스를 제공하는 서버

어댑티브 비트레이트 알고리즘: 네트워크 상태에 따라 해상도를 조절하는 알고리즘

연합 학습: 데이터를 중앙 서버로 모으지 않고 분산된 기기나 로컬 데이터로 모델을 개별 학습 -> 가중치만 중앙 서버로 보내 통합하는 것

스파게티 코드: 구조화되지 않고 복잡하게 얽혀서 유지보수가 맹우 어려운 컴퓨터 소스 코드

주니어 개발자의 생각하는 근육 퇴화:
- 고민하지 않고 AI에게 정답만 요구하면 문제 해결 능력 퇴화

TDD: 이 기능은 이렇게 동작해야 됨이라는 기준을 세움 AI가 짠 코드가 그 테스트를 통과하는지 검증하는 절차를 강제해야 

코드리뷰 에이전트 활용: AI를 코드 생성용으로만 사용 x 내가 짠 코드의 취약점이나 리펙토링 제안 용도로 활용

변이 테스트 도입: 테스트 코드 자체가 튼튼한지 검증하기 위해 소스코드를 임의로 고장 내보는 것

휴먼 아키텍트 원칙: 구현은 AI가 설계와 통신 규약은 개발자가 정의

사일로 Silo: 조직 내 부서 간의 장벽이나 정보 공유 단절, 부서 이기주의

멀티모달 AI 에이전트: 타자를 치는 것조차 힘든 고령층을 위해 말로 하면 알아서 해주는 인터페이스

RAG(검색 증강 생성): 데이터베이스 내에서만 정보를 참조

마스킹: 고객데이터는 암호화하여 저장

생태계의 파편화: 각기 다른 요소를 개발자가 모든 기기와의 호환성을 보장하기 어려워지는 현상

오프로딩: 모바일 장치나 엣지 디바이스의 컴퓨팅 자원을 보완하기 위해 서버같은 원격 고성능 컴퓨팅 자원으로 보내 효율성을 높이는 기술

반응형공간 UI: 개발자가 한 번 코드를 짜면 기기에 맞게 화면을 맞춰주는 기술
