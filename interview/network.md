# 2. 네트워크
## OSI 7계층
- 네트워킹 표준을 7개의 층으로 나눈 것
- 통신 중 문제 발생 시 단계별로 파악하여 신속한 해결이 가능
- 암기 팁: '**물데네전세표응**' (물리, 데이터 링크, 네트워크, 전송, 세션, 표현, 응용)

### [각 계층의 역할]

1. **물리 계층**: 전기적 신호 전달
2. **데이터 링크 계층**: 인접한 기기 간 통신
3. **네트워크 계층**: IP를 사용하여 최적의 경로 결정 (라우팅)
4. **전송 계층**: 데이터 송수신 및 신뢰성 보장
5. **세션 계층**: 통신 세션 관리 및 동기화
6. **표현 계층**: 데이터 형식 변환 및 암호화 (규칙 일치)
7. **응용 계층**: 사용자가 사용하는 프로그램 영역

```
OSI 7계층은 인터넷 통신 표준으로, 효율적인 통신과 문제 해결을 위해 물리부터 응용 계층까지 총 7개 층으로 나뉩니다. 면접 시에는 각 계층의 명칭과 핵심 역할(예: 네트워크 계층은 IP와 경로 결정 등)을 연결하여 설명하는 것이 중요합니다.
```

## TCP vs UDP
- 전송 계층(L4)에서 사용하는 주요 프로토콜
### [TCP (Transmission Control Protocol)]
- **특징**: 높은 신뢰성, 데이터 전달 보장, 1대1 통신
- **제어**:
    - **흐름 제어**: 수신자의 처리 능력을 고려해 데이터량 조절
    - **혼잡 제어**: 네트워크 혼잡을 방지하기 위해 데이터 전송 제한
- **연결**: 3-way handshake(설정), 4-way handshake(해제)

### [UDP (User Datagram Protocol)]
- **특징**: 비연결형, 빠른 속도, 데이터 손실 가능성 있음
- **연결**: 1대1, 1대다, 다대다 통신 가능
- **용도**: 실시간 스트리밍(유튜브 라이브, 트위치 등)

```
TCP는 신뢰성 있는 통신을 위해 3-way handshake를 통한 연결 설정과 흐름/혼잡 제어 기능을 갖춘 프로토콜입니다. 반면 UDP는 연결 지향이 아니어서 신뢰성은 낮지만 속도가 빨라 실시간 스트리밍 서비스에 적합합니다.
```

### IP (Internet Protocol)
- 인터넷 망을 통해 패킷(데이터 단위)을 전달하는 규칙
- IP 주소는 네트워크 상에서 컴퓨터의 위치(논리적 주소)를 의미

### [IP의 주요 특징]
- **비연결성**: 패킷을 보낼 때 경로를 미리 정하지 않음
- **비신뢰성**: 데이터 전달을 완벽히 보장하지 않음
- **보완**: ICMP 프로토콜을 사용하여 전송 오류 시 재전송 요청 가능

```
IP는 인터넷 망에서 패킷을 전달하는 프로토콜로, 비연결성과 비신뢰성이라는 특징을 가집니다. 라우터를 통해 데이터를 전달하며 위치 정보인 IP 주소를 기반으로 작동합니다.
```

## DNS (Domain Name System)
- 사람이 이해하기 쉬운 도메인 주소를 컴퓨터용 IP 주소로 변환하는 시스템
- **작동 순서**: URL 입력 → ISP DNS 해석기 → 루트 서버 → 하위 DNS 서버 → 최종 IP 획득

```
DNS는 naver.com 같은 도메인 이름을 실제 IP 주소로 변환해주는 시스템입니다. 사용자가 URL을 입력하면 여러 단계의 서버(루트, 하위 서버 등)를 거쳐 IP를 찾아오는 방식으로 작동합니다.
```

## 로드 밸런서 (Load Balancer)
- 서버에 가해지는 부하를 여러 대의 서버로 분산시키는 시스템
- **L4 로드 밸런싱**: 전송 계층 정보(IP, 포트)를 기반으로 분산
- **L7 로드 밸런싱**: 응용 계층 정보(URL, 쿠키, HTTP 헤더)를 기반으로 분산

```
로드 밸런서는 서버 부하를 분산하여 서비스 안정성을 높입니다. L4는 IP와 포트 정보를 보며, L7은 애플리케이션 수준의 패킷 내용까지 확인하여 DDoS 방어와 같은 정교한 트래픽 필터링이 가능합니다.
```

- 면접에서는 자신이 확실히 이해하지 못한 어려운 용어는 사용하지 않는 것이 좋음
